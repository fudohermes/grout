# AGENTS.md — Grout

> Working notes for AI agents and humans working on this repo.
> For the design brief, see [GROUT.md](GROUT.md).

## What this is

Grout is a small, purpose-built **filler and interstitial media store** for
the Fudo stack. It holds bumpers, idents, and (later) orphan long-form clips
that don't fit Jellyfin's metadata-provider worldview.

Key properties:

- **Flat structure** — no collections, no hierarchy. Tags only.
- **Tag + duration filters** — `?channel=X&tags=daytime,fun,kids&min_ms=65000&max_ms=90000&random=true&limit=5`
- **Content-addressed storage** — dedup by SHA-256 of source bytes
- **Owns its own metadata** — name/description/tags are freely mutable
- **Streams to Pseudovision by file path** (PV is co-mounted on `arr-data`)
  with an HTTP byte-range fallback
- **AI enrichment via Tunabrain only** — no model credentials in Grout

The name: *grout fills the gaps between tiles. Grout fills the gaps between
programs.* See `GROUT.md §1`.

## How it fits in the ecosystem

```
   Tunarr Scheduler  ──►  Grout     ◄──── Pseudovision
   (writes bumpers,        │                (reads at air time,
    generated by Tunabrain)│                 streams by path or
                           │                  HTTP byte-range)
                           ▼
                       Tunabrain (LLM enrichment)
                           ▲
                           │
                         Marquee (UI)
```

- **Pseudovision** is the primary read consumer. At air time, it queries
  Grout for gap-appropriate filler and streams the result. The
  `path` field in responses is the primary stream path (co-mounted file
  system); the HTTP `/stream` endpoint is the fallback.
- **Tunarr Scheduler** is the primary writer for generated bumpers. Since
  commit `8df074d` (Jul 2026) the bumper pipeline stages to Grout instead
  of Jellyfin. The ffmpeg compose step runs in Tunarr Scheduler; Grout
  receives the finished file via `POST /grout/media`.
- **Tunabrain** is invoked for metadata enrichment (`POST /grout/media/:id/enrich`).
  Grout passes the source-file content and asks Tunabrain for tags, name,
  and description. Grout holds no model keys; the Tunabrain client is
  configured by URL.
- **Marquee** is the UI for browsing and managing filler (read + write on
  the standard CRUD endpoints).
- **Jellyfin** is explicitly *not* the upstream for Grout. Grout owns
  everything that doesn't fit Jellyfin's worldview — synthetic + orphan
  web content.

```bash
# Migrate the DB
clojure -X:migrate

# Run (default config: resources/config.edn)
clojure -M:run

# Tests (Kaocha)
clojure -M:test
clojure -M:test --focus grout.http.media-test

# nREPL
clojure -M:repl

# Lint
clojure -M:lint
```

Tooling:

- **Clojure CLI** (deps.edn-based)
- **Java 21+**
- **ffmpeg + ffprobe** on `$PATH` (used at intake to probe + normalize)
- **PostgreSQL** with a writable schema; the `GROUT_DB_URL` env var points
  at it
- **Filesystem** writable at the `GROUT_MEDIA_DIR` mount (co-mounted with PV
  on `arr-data` in production)

## Public API surface

| Method & path | Purpose |
|---|---|
| `POST /grout/media` | Intake a file on the mount: hash → probe → normalize → insert. Dedups by content hash (`201` new, `200` matched/retagged/revived) |
| `GET /grout/by-hash/:hash` | Look up an item by SHA-256 of the source bytes (CLI pre-check) |
| `GET /grout/media` | Query by `channel`, `tags` (AND), `min_ms`/`max_ms`, `kind`, `random`, `limit` |
| `GET /grout/media/:id` | Fetch one |
| `PATCH /grout/media/:id` | Mutate `name`/`description`/`tags`/`channel` |
| `DELETE /grout/media/:id` | Soft-delete (supersede); `?hard=true` unlinks the file |
| `GET /grout/media/:id/stream` | Byte-range HTTP streaming fallback (`Range` → `206`) |
| `GET`/`POST /grout/media/:id/tags` | List / add tags |
| `POST /grout/media/:id/enrich` | Trigger Tunabrain metadata enrichment |
| `GET /health` | Health/readiness |
| `GET /api/version` | Build info |
| `GET /openapi.json`, `/swagger-ui` | API docs |

**Channel semantics:** a `channel=X` query matches that channel **or**
generic (null-channel) items, which are usable on any channel (per
`GROUT.md` §14). Response bodies use kebab-case keys (`duration-ms`,
`stream-url`) per the service-wide JSON convention.

## Common pitfalls

1. **Service may be in scaffolding-only state.** The empty-body `/health` +
  `/api/version` + `/openapi.json` pattern means Reitit is up but no
  business routes are registered. This is the most common "is Grout
  broken?" report and the answer is almost always a build/wiring problem,
  not a code bug. See the caveat in **Live endpoints** above and the
  `pseudovision-ecosystem-development` skill's "Empty OpenAPI Spec
  Diagnosis" pattern.
2. **Stream by path first, HTTP fallback second.** PV is co-mounted on
  `arr-data`; queries return a `path` field so PV can stream the file
  directly. Don't reach for `/grout/media/:id/stream` unless the caller
  isn't on the same mount — using it as the primary path adds latency and
  byte-range overhead. (Core principle #2 in `GROUT.md`.)
3. **No collections, no hierarchy.** The temptation to add folders /
  seasons / series is the most common design drift. Don't. Tags only.
  (Core principle #5.)
4. **No model credentials in Grout.** The Tunabrain client is configured
  by URL (`TUNABRAIN_URL`); the LLM key lives in Tunabrain or its
  upstream. If you find yourself adding an `OPENAI_API_KEY` env var to
  Grout, stop — extend Tunabrain's API instead. (Core principle #3.)
5. **Intake is normalize-once, not transcode-on-read.** Every item is
  ffprobed and (if off-profile) transcoded to PV's playout profile before
  it's queryable. If a new item is invisible in queries, check
  `/var/log/grout/intake.log` (or whatever the env-configured log path is)
  for ffprobe errors — most failures are off-profile files that the
  transcoder choked on.
6. **Soft-delete by default.** `DELETE /grout/media/:id` marks the row as
  superseded; it doesn't unlink the file. Use `?hard=true` to actually
  unlink. If you're cleaning up storage, you need the hard-delete form
  *and* a follow-up garbage collection pass for orphans.
7. **Channel=null is a feature, not a bug.** Generic items (channel NULL
  in the DB) are usable on any channel. Don't filter them out by
  default — they're the "filler that fits anywhere" tier.
